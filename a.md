## ۸. بازبینی کدبیس تیم + اصلاحات (۱۷ جولای، بعدازظهر)

تیم فازهای ۰-۶ را ساخته بود: backend ماژولار FastAPI (۱۴ ماژول)، وب Next.js (۹ صفحه)، سیستم امتحان کانکور + بانک ۸۸ سوال + چت AI. **معماری را درست رعایت کرده‌اند** (router/service/repository، الگوی ۵ فایلی، توکن‌های دیزاین). بازبینی کامل انجام شد:

### مشکلات پیدا شده

| # | مشکل | شدت |
|---|---|---|
| ۱ | **حفره امنیتی:** همه POST/PATCH های محتوا بدون لاگین باز — ردیف آشغال `fghjk` در DB اثباتش | 🔴 بحرانی |
| ۲ | **پنل ادمین فقط-خواندنی:** بدون فورم افزودن/ویرایش/حذف، بدون لاگین (شکایت ثبت‌شده کاربر) | 🔴 |
| ۳ | **سرور کهنه روی پورت 4000:** پروسه next-server قدیمی، بیلد کهنه سرو می‌کرد — دلیل «باز نشدن» پنل | 🔴 |
| ۴ | **باگ JWT:** payload فقط sub+exp → دو لاگین در یک ثانیه = توکن یکسان = خطای 500 | 🟠 |
| ۵ | دیتای آشغال + پوهنځی CS واقعی زیر پوهنتون آشغال `fghjk` ثبت شده بود | 🟠 |
| ۶ | ۳۷/۴۰ تست شکست (event loop مشترک engine در conftest) | 🟠 |
| ۷ | **باگ آزمون رشته:** نتیجه به جای slug/نام رشته، UUID برمی‌گرداند — لینک نتایج خراب | 🟠 |
| ۸ | هیچ DELETE endpoint نبود | 🟡 |

### اصلاحات انجام شده (همه تست شده)

1. **امنیت:** همه نوشتن‌ها پشت `get_current_admin` + DELETE برای پوهنتون/پوهنځی/دیپارتمنت/خبر/FAQ/رویداد + محافظت reindex — تست: POST بدون توکن → 401 ✅
2. **JWT:** `jti` تصادفی (admin_auth + users)
3. **دیتابیس:** CS به پوهنتون هرات برگشت؛ `fghjk` و `faculty-tets` حذف → ۱ پوهنتون، ۱۶ پوهنځی، ۷۵ دیپارتمنت
4. **پنل ادمین بازسازی کامل:** `/admin/login` + لایه `adminApi.ts` (توکن + refresh خودکار) + CRUD کامل با فورم مودال (پوهنځی/دیپارتمنت/خبر/FAQ/رویداد) + دکمه «بازسازی دانش چت‌بات» + مدیریت خطا — دیگر crash نمی‌کند
5. **حالت تک-پوهنتون:** تب پوهنتون‌ها از UI حذف (با کامنت راهنمای برگرداندن)؛ معماری multi-tenant دست‌نخورده
6. **آزمون رشته:** join با جدول رشته‌ها → slug و نام واقعی + فقط فارغ‌ده‌ها
7. **تست‌ها:** conftest با engine per-test + NullPool → **۴۰/۴۰ سبز**
8. **E2E تایید شد:** لاگین → ساخت FAQ → نمایش در API عمومی → ویرایش → حذف ✅؛ همه صفحات 200؛ پروسه کهنه کشته شد و بیلد نو سرو می‌شود
