# فاز ۴ — اتصال Groq + مدیریت Provider (۱ روز) — درخواست ۲

کلید در `assist/13-groq-api-key.txt` موجود است. معماری Provider از قبل آماده است — فقط یک کلاس جدید:

- [ ] ۴.۱ فایل `backend/app/providers/ai/groq.py`:

```python
import httpx
from app.config import settings
from .base import AIProvider

class GroqProvider(AIProvider):
    """Groq — سازگار با فورمت OpenAI؛ بسیار سریع، سطح رایگان دارد"""
    async def chat(self, system, messages, max_tokens=800):
        async with httpx.AsyncClient(timeout=60) as client:
            r = await client.post(
                "https://api.groq.com/openai/v1/chat/completions",
                headers={"Authorization": f"Bearer {settings.ai_api_key}"},
                json={
                    "model": settings.ai_model,   # مثلا: llama-3.3-70b-versatile
                    "max_tokens": max_tokens,
                    "messages": [{"role": "system", "content": system}, *messages],
                },
            )
            r.raise_for_status()
            return r.json()["choices"][0]["message"]["content"]
```

- [ ] ۴.۲ در `providers/ai/factory.py`: شاخه `case "groq": return GroqProvider()`
- [ ] ۴.۳ در `.env`: `AI_PROVIDER=groq` + `AI_MODEL=llama-3.3-70b-versatile` + کلید از فایل assist (⭐ کلید فقط در `.env` — فایل assist از Git خارج بماند و بعد از انتقال حذف شود)
- [ ] ۴.۴ تست دستی: سوال دری به `/ai/chat` → جواب معنادار
- [ ] ۴.۵ داشبورد ادمین: نمایش provider و model فعال (خواندنی — endpoint کوچک `GET /admin/ai/status`)

**⚠️ نکته تصمیم‌گیری:** مدل‌های Groq (خانواده Llama) در دری/پشتو از Gemini ضعیف‌ترند. استراتژی پیشنهادی: **Groq برای توسعه/تست (رایگان و بسیار سریع) — Gemini برای production.** تعویض فقط یک خط `.env` است؛ هر دو provider در کد بمانند. بعد از راه‌اندازی، ۱۰ سوال دری یکسان را با هر دو تست کنید و کیفیت را خودتان مقایسه کنید.

**معیار پذیرش فاز ۴:** با `AI_PROVIDER=groq` چت دری کار کند؛ برگرداندن به `gemini` بدون تغییر کد.
